# Project Overview - Magisterium

## 1. Project Identity
Magisterium is a text-based anime-style RPG web game.

The game is inspired by:
- SAO-style progression
- Danmachi-style dungeon growth
- isekai / reincarnation / farming progression fantasy

The goal is not to build a full MMORPG immediately.
The goal is to build a strong RPG system foundation first, then expand gradually.

---

## 2. Core Design Philosophy
The project should follow these principles:

- Start with a small but meaningful playable system
- Keep data structures clean from the beginning
- Separate game rules from UI logic
- Make each class feel different in identity
- Allow future hybrid builds without breaking the core system
- Avoid hard-coding weapon type = one fixed stat forever

---

## 3. System Priorities
The current priorities are:

1. Character Creation
2. Core Stat System
3. Class Identity
4. Future Combat Foundation
5. Equipment / Skills / Quest expansion later

---

## 4. Long-Term Development Direction
Planned future systems may include:
- battle system
- monsters
- inventory
- equipment
- skill upgrades
- quests
- dungeons
- guild / chat / social features

---

## 5. Current Development Rule
During the early phase:
- keep scope small
- make the data model solid
- do not overbuild combat too early
- save important design decisions inside the repository docs
